## Completed: Next.js 15 params migration

### What changed
Next.js 15 made route-handler `params` (and `searchParams`) into Promises. Every handler that destructured `params` synchronously is a latent bug under Next 15 — TS now flags it; runtime returns undefined. Migrated all 17 handlers in one mechanical pass before any new routes (deliverable endpoints up next) can ship the wrong pattern.

### Files migrated (17)
- `app/audits/[auditId]/page.tsx`
- `app/api/audits/[auditId]/route.ts`

- `app/api/audits/[auditId]/risk-summary/approve/route.ts`
- `app/api/audits/[auditId]/questionnaire/route.ts` (GET + POST)
- `app/api/audits/[auditId]/questionnaire/approve/route.ts`
- `app/api/audits/[auditId]/questionnaire/responses/route.ts`
- `app/api/audits/[auditId]/questionnaire/addenda/route.ts`
- `app/api/audits/[auditId]/questionnaire/status/route.ts`
- `app/api/audits/[auditId]/questionnaire/vendor-return/route.ts`
- `app/api/audits/[auditId]/trust-assessment/route.ts` (GET + POST + PATCH)
- `app/api/audits/[auditId]/vendor-service/route.ts` (GET + POST)
- `app/api/audits/[auditId]/vendor-service/mappings/route.ts`
- `app/api/audits/[auditId]/vendor-service/mappings/[id]/route.ts`
- `app/api/protocols/[protocolVersionId]/risk-objects/route.ts` (GET + POST)
- `app/api/protocols/[protocolVersionId]/risk-objects/[id]/route.ts`

### Pattern applied uniformly
```ts
// Before
export async function GET(
  req: NextRequest,
  { params }: { params: { auditId: string } }
) {
  // …uses params.auditId…
}

// After
export async function GET(
  req: NextRequest,
  { params }: { params: Promise<{ auditId: string }> }
) {
  const { auditId } = await params;
  // …uses auditId…
}
```

### Side wins
- 3 incorrect 204 responses (`NextResponse.json(null, { status: 204 })`) corrected to `new NextResponse(null, { status: 204 })` — 204 must have empty body. Found while touching `risk-summary` GET, `questionnaire` GET, `trust-assessment` GET, `vendor-service` GET.

`grep -rn "params: { " app/` returns zero results. `grep -rn "params\." app/api` returns nothing outside `Promise<>` annotations or destructuring statements.

  - `approveRiskSummary` refuses empty/whitespace narrative; stamps approvedAt/approvedBy when valid
  - `updateRiskSummary` demotes APPROVED → DRAFT (clears approvedAt/approvedBy); stays DRAFT when editing draft (no demotion noise); writes a delta with the demotion reason
  - `getRenderedRiskSummary` returns null on missing; returns ISO date strings + protocolRiskRefs array
- **`tests/lib/deliverables.test.ts`** — 12 tests
  - `it.each(DELIVERABLE_KINDS)` runs the same scenarios across all three kinds: stub creates DRAFT, getRenderedDeliverable returns null on miss
  - ALREADY_EXISTS / AUDIT_NOT_FOUND error codes
  - Per-kind validity for approval: empty letter body, agenda items with whitespace topic, checklist items with empty prompt
  - Demote-on-edit: APPROVED → DRAFT
  - INVALID_CONTENT_SHAPE on cross-kind content (letter shape PATCHed to agenda); error carries Zod issues
  - RenderedDeliverable carries the `kind` discriminator
- **`tests/lib/questionnaires-approve.test.ts`** — 4 tests
  - Refuses approval when no instance exists
  - `it.each` runs every non-COMPLETE status: refused (5 cases collapsed)
  - Approves + stamps when COMPLETE
  - Writes delta on approval

### Design choices
- **Integration over unit**: every library function tested runs inside `prisma.$transaction` and writes deltas. Mocking would test the mocks. Real DB exercises FK constraints + transaction semantics + delta tracking together.
- **Single-fork concurrency**: tests share one physical DB; per-worker schemas would be over-engineering at this volume. Re-evaluate if test count or runtime grows.
- **Auto-discovered truncation**: schema additions never break the harness. New models truncate automatically.
- **State-shortcut factories**: tests of library X use real X for the operation under test, but setup of upstream artifacts uses direct prisma.create. Keeps each test focused on one surface.

## Completed: Code review — critical issue fixes

### Issues addressed (from engineering:code-review)

**Critical 2 — `captureVendorReturn` N+1 inside transaction** (`lib/questionnaires.ts`)
- Before: per-response `findUniqueOrThrow` loop → 3N sequential queries (read + update + delta) inside one `$transaction`. 60-question vendor return = 180 queries holding transaction locks.
- After: single `findMany` batch-reads all existing response rows into a `Map<questionId, row>` before the loop. Fail-fast validation (throw if any questionId missing) before any writes. Per-row `update` + `writeDelta` unchanged — these can't be batched due to differing `responseText` per row. Net: 1 + 2N queries instead of 3N; read lock time eliminated.
- Bonus: `respondedAt = new Date()` promoted outside the loop — all responses in a bulk return share one timestamp, which is the correct semantics.

**Critical 3 — `GET /api/history` without audit scope** (`app/api/history/route.ts` → `app/api/audits/[auditId]/history/route.ts`)
- Before: global `/api/history?objectType=X&objectId=Y` — any caller with a valid UUID could read any object's history across all audits; no ownership boundary.
- After: moved to `/api/audits/[auditId]/history?objectType=X&objectId=Y` — URL scoped to an audit, consistent with all other audit resource routes, inherits auth middleware when session lands.
- Old route returns `410 Gone` with a clear redirect message for any stale clients.
- `HistoryDrawer` gained `auditId: string` prop; fetch URL updated.
- Both call sites updated: `RiskSummaryPanel` (already had `auditId`) and `PreAuditDraftingWorkspace > DeliverablePane` (already had `auditId`).
- Phase 1 note: `auditId` is URL-validated (Prisma params); full cross-resource ownership check (verify `objectId` ∈ `auditId`'s objects) lands with session auth.

**Suggestion 2 (consistency) — `responses/route.ts` manual JSON parse** (`app/api/audits/[auditId]/questionnaire/responses/route.ts`)
- Before: rolled its own `try/catch` JSON parse + `ZodError` handling — inconsistent with every other D-010+ route.
- After: migrated to `parseJson` + `validationError` helpers. Body reduced from 55 lines to 37.

### Deferred (external blocker)

Fix requires session auth (provider decision pending). All routes are `TODO`-commented. No build action possible until auth provider is decided.
